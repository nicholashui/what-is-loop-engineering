咁我哋而家就由零件講到郁動。上一章我哋打開部機器,逐一講咗佢八個部分嘅名。而家我哋要令佢哋轉動起嚟。一張組件清單話畀你知一個 Loop 係由咩組成。而個生命週期就話畀你知,呢啲部分隨住時間係點樣互相配合。佢話畀你知,控制權係按咩次序由一個部分傳去下一個部分,亦都話畀你知,每個部分喺繞圈嘅每一程入面貢獻咗啲咩。

個生命週期,其實就係一條好簡單問題嘅答案。一個 Loop 運行嗰陣,一步一步,究竟實際發生咩事?當你親手去提示一個 AI,嗰個次序係你自己提供。你決定有嘢要做。你叫個代理去做。你睇結果。你判斷佢做得好唔好。你記住自己做到邊。然後你決定要唔要再叫多次。一個 Loop 就係將呢個完全一樣嘅次序,編碼成一個會重複嘅循環,咁佢就可以喺你唔使坐喺度嘅情況下運行。我哋接住要逐個行過嘅階段,正正就係嗰啲判斷,只不過將佢哋講到明明白白,並且排好次序。

一個 Loop 會將工作推過七個有次序嘅階段。第一到第六個階段,組成一次迭代嘅主體,而第七個就係結束呢次迭代、並且揀定下一步嘅嗰個決定。每一個階段,都係由上一章嗰一個或者幾個組件去負責。

第一階段,係發現或者觸發工作。當工作進入個 Loop,循環就開始。可能係個 Loop 發現有嘢要做,例如清單上下一項未完成嘅嘢;又或者係一件外部事件去觸發佢。呢個係自動化同觸發,即係 Automation and Trigger 組件嘅職責。佢可能係一個排好時間、半夜啟動嘅排程工作;一個喺新提交嘅 issue 上面嘅 webhook;又或者係一條 slash-loop 式嘅指令。喺呢個階段啟動迭代之前,咩都唔會發生;而且呢度唔需要有人手動打一句「開始」。

第二階段,係喺一個隔離嘅工作空間入面生成代理。手頭有咗工作,個 Loop 就會整一個乾淨、密封嘅地方畀件工作去發生,並且喺入面啟動一個或者幾個代理。呢個階段用到工作樹同隔離,即係 Worktrees and Isolation,亦即係一份獨立嘅程式碼複本同埋一條專用嘅分支,咁無論個代理做咩,都會被困住、亦都可以還原。被生成出嚟嗰個代理,就係生成代理,即係 Generator Agent;當佢開始嗰陣,佢會收到專案嘅技能同知識,即係 Skills and Knowledge,亦即係啲編碼規範同埋代理指引檔案,咁佢由第一個動作開始,就已經跟住你嘅規則去做。

第三階段,係代理動手產生輸出。喺佢嗰個隔離嘅工作空間入面,Generator Agent 親手做嘢。佢讀個任務,佢改個程式碼庫,佢產生一組改動。當佢需要接觸外面嘅世界,例如由一個原始碼控制主機讀程式碼,又或者由一個問題追蹤系統攞細節,佢就會透過連接器,即係 Connectors 去做。呢個係唯一一個真正有新輸出被創造出嚟嘅階段。其他每一個階段嘅存在,都係為咗啟動佢、檢查佢、記住佢,又或者決定點樣處理佢。

第四階段,係評估器檢查輸出。啱啱產生出嚟嘅輸出,而家會由外面去判斷。評估器,即係 Evaluator,佢嘅角色係一個評審或者驗證者;佢會行成個測試套件、審查啲改動,又或者按一份評分準則去評分,然後畀返一個裁決。究竟呢樣夠好,定係唔夠好?呢個階段保護個 Loop,唔好好自信咁接受咗錯嘅或者低質素嘅工作。而最關鍵嘅一點係,個評審同個做嘢嘅係分開嘅。Generator Agent 唔可以批改自己份功課。

第五階段,只會喺輸出唔夠好嗰陣先發生。喺嗰個情況,個 Loop 唔會將件工作掉咗。佢會由失敗入面學嘢。究竟錯咗咩嘅實際信號,無論係失敗嘅測試輸出、審查者嘅評語,定係評分準則上嗰啲低分備註,都會作為額外脈絡送返畀 Generator Agent,然後循環就返去第三階段,再試多次。呢條反饋路徑,正正就係令一個 Loop 成為一個 Loop 嘅核心。每一次被否決嘅嘗試,都令下一次有更好嘅資訊基礎,而唔係盲目咁重複。而呢個階段係有條件嘅。當輸出已經足夠,個 Loop 就會直接跳過。

第六階段,係儲存狀態。無論今次迭代係完全成功,定係只係取得部分進展,個 Loop 都會喺任何單一代理嘅脈絡以外,記低發生咗咩事,用嘅係狀態同記憶,即係 State and Memory 組件。佢可能會更新一個待辦檔案、喺一個草稿簿寫低筆記,又或者喺一個專案看板上面移動一張卡。呢樣嘢令到下一次迭代可以喺今次嘅基礎上繼續,而唔使由零開始。亦都令到你隨時都可以讀到件工作目前嘅狀態。

第七階段,係嗰個決定。當迭代完成、狀態亦都儲咗起嚟,個 Loop 就到達佢嘅決定點。主要由停止條件,即係 Stopping Condition 去引導,佢會喺三條路入面揀一條。佢可以繼續去做多一次迭代;佢可以生成一個 sub-agent,即係子代理,去處理一件獨立嘅工作;又或者佢可以停低。呢個三選一嘅選擇,重要到值得我哋單獨拎出嚟睇。

如果咁樣有幫助,你可以將成個次序諗像成一條流程:工作喺頂部入嚟,每逢輸出唔夠好就喺入面嗰條重試路徑兜返圈,最後到達底部嗰個決定點。不過,我哋啱啱行過嘅嗰啲步驟,本身就係嗰條流程嚟,所以我哋集中講吓嗰個決定。

每一次繞個 Loop,都係喺同一個分岔位結束;而佢揀邊條路,係由清晰嘅準則決定,而唔係靠估。Stopping Condition 喺呢度係主要嘅權威,但呢個決定的的確確係三選一。

第一個選項,係停止。當兩種規則其中一種被觸發,個 Loop 就會結束。第一種係成功。目標已經達到。Evaluator 嘅測試通過咗,又或者評分準則嘅分數已經去到佢嘅門檻。件工作做完,所以個 Loop 就有意識咁完結,並且交付佢嘅成果,例如透過一個 Connector 開一個 pull request。第二種係安全。一個防護欄被觸碰到。迭代次數去到上限,又或者代幣同成本嘅預算用晒。喺呢度,個 Loop 喺未成功嘅情況下停低,並且升級交畀一個人去處理,而唔係喺度空轉落去。呢條規則,就係防止一個 Loop 變成一個失控過程嘅嘢。

第二個選項,係繼續。如果成功規則同安全規則都未被觸發,咁就即係仲有有用嘅工作要做,而且仲有預算去做,所以個 Loop 就再兜多一圈。喺實際操作上,繼續就係喺第五階段嘅反饋之後所揀嘅路。目前嘅任務未完成,嘗試次數同預算都喺界限之內,所以個 Loop 就用佢啱啱學到嘅嘢,喺同一個任務上開始一次全新嘅迭代。

第三個選項,係生成一個 sub-agent。有陣時最啱嘅做法,既唔係重試同一個任務,亦唔係放棄,而係去委派。當件工作自然咁分裂成一件獨立、自成一體嘅嘢,一件夠大、值得有自己嘅隔離工作空間同自己評估嘅聚焦子任務,個 Loop 就會生成一個 sub-agent 去處理佢。呢個 sub-agent 會運行佢自己一個迷你嘅生命週期,有自己嘅生成、動手、評估同儲存步驟,並且將佢嘅成果回報返畀母 Loop。傾向去生成嘅準則有兩個:一個係可分解性,即係件任務可以乾淨咁分開;另一個係獨立性,即係嗰件嘢可以喺唔使不斷協調嘅情況下做。當一個代理逐條線性咁慢慢磨,會比起將工作分派出去更慢、更亂嘅時候,就揀呢條路。

所以,簡單講。做完咗或者預算用晒,就停。當件工作應該被分割同委派,就生成一個 sub-agent。當手頭上嘅任務仲有嘢要做、亦都仲有資源去繼續,就繼續落去。

值得留意嘅係,呢度其實有兩個 Loop 喺度運作。有一個內層 Loop,即係第三到第五階段,喺嗰度一個單一任務會帶住反饋不斷重試,直到 Evaluator 接受佢,又或者一個安全上限被觸發為止。亦有一個外層 Loop,即係第一到第七階段,佢由一個工作單位移去下一個。第七階段嘅決定,就係 Stopping Condition 居住嘅地方。佢係唯一一個揀繼續、生成,定係停止嘅位置;亦都係令到成個結構會結束、而唔會永遠運行落去嘅原因。

而家,個生命週期並唔係咩另外一個機制,夾硬裝喺啲組件上面。佢本身就係啲組件,只係喺郁動之中被觀察到。每一個階段,純粹就係某一個組件做佢本份工作嗰一刻。第一階段,發現或者觸發工作,就係 Automation and Trigger 組件。第二階段,喺隔離之中生成代理,就係將 Worktrees and Isolation、Generator Agent、同埋 Skills and Knowledge 集合埋一齊。第三階段,代理動手產生輸出,就係 Generator Agent 同 Connectors 並肩工作。第四階段,檢查輸出,就係 Evaluator。第五階段,將反饋送返並且重複,就係 Evaluator 將佢嘅裁決交返畀 Generator Agent。第六階段,儲存狀態,就係 State and Memory 組件。而第七階段,決定究竟係繼續、生成定停止,就係 Stopping Condition,並且有 Connectors 喺旁邊隨時準備好做交付。

由頂到底咁追蹤一次,你就跟過咗一份工作,由准許佢入嚟嗰個觸發,一路去到放佢走嗰個決定。呢個正正就係上一章預告嗰段旅程,嗰陣佢話,八個部分追蹤緊一份工作單位穿過個 Loop 嘅路徑。啲組件係名詞,而個生命週期就係動詞。兩樣都喺手,你就準備好迎接下一章,喺嗰度你會建造一個真正運行呢個循環嘅 Loop,由一個最基本嘅版本開始,逐步演化成一個生產級別嘅系統。

咁以下就係幾個重點。一個 Loop 係以一個有次序、會重複、共七個階段嘅生命週期去運行。第一,發現或者觸發工作。第二,喺一個隔離嘅工作空間生成代理。第三,代理動手產生輸出。第四,Evaluator 檢查輸出。第五,如果唔夠好,就將反饋送返並且重複。第六,儲存狀態。第七,決定係繼續、生成一個 sub-agent,定係停止。

第三到第五階段,組成一個內層嘅反饋循環。一次被否決嘅嘗試唔會被丟棄。佢會作為額外脈絡返畀 Generator Agent,咁每一次重試都比上一次有更好嘅資訊基礎。

第七階段嘅決定點係三選一。個 Loop 會喺成功嗰陣停止,即係當測試通過,又或者評分準則嘅門檻達到;又或者佢會喺一個安全上限嗰陣停止,即係當迭代次數到頂,又或者預算用晒,並且升級交畀一個人。當任務未完成而且仲有資源,佢就繼續。而當件工作夠可分解、亦都夠獨立,可以作為佢自己一個迷你生命週期去委派,佢就生成一個 sub-agent。

Stopping Condition 喺決定點係唯一嘅權威。佢就係保證個 Loop 會有意識咁結束,而唔係永遠運行落去嘅嘢。

最後,每一個生命週期階段,純粹就係八個組件其中一個做緊佢本份工作。啲組件係嗰啲部分,而個生命週期,就係嗰啲部分喺郁動之中。
