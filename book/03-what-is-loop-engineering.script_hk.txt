最後，就係自動咁運作。一旦你設計好咗，個系統就會自己行呢個循環，唔需要有人去逐句寫指令。

呢種做法所製造出嚟嘅產物，亦都有佢自己嘅名。一個 Loop，就係由一個 loop engineer（迴圈工程師）所建立嘅統籌系統。佢會發掘工作、生成 agent、驗證佢哋嘅輸出、保存狀態，並且決定要繼續定係停低。喺成本書入面，Loop Engineering 係指呢套學問，亦即係一種思考同設計嘅方式；而 Loop 就係你所建立嘅嗰件嘢。將呢兩樣分清楚，本書其餘部分就會保持清晰：你做嘅係 Loop Engineering，而你最後得到嘅就係一個 Loop。

呢度有一個澄清，可以避免一個常見嘅混淆。Loop 並唔係 agent。嗰個 AI coding agent，亦即係本書所講嘅 Generator Agent（生成代理），即係真正去產生程式碼同改動嘅嗰個部件，佢只不過係 Loop 嘅其中一部分。Loop 係周圍嘅機制，由佢去決定 agent 應該做咩、去運行佢、去評斷結果，再不斷重複。當本書講「Loop 決定」或者「Loop 驗證」嘅時候，意思係指嗰套周圍嘅機制，而唔係入面嗰個 agent。

而家我哋睇下手動提示同 Loop 之間嘅對比。要理解 Loop Engineering，最快嘅方法就係將佢直接同佢所取代嘅嘢擺埋一齊比較。

諗下手動、逐個回合咁提示係點運作。當你用手去提示一個 AI coding agent 嗰陣，你自己就係嗰個控制系統。諗下喺一個工作環節入面你實際上做緊啲咩。你將個目標記喺腦入面，你知道「做完」係咩樣。你打一段提示，描述下一步要做咩。你睇 agent 嘅輸出，再判斷佢係咪夠好。如果係錯嘅，你就寫一段更正，再提示多次。你會記住已經試過咩，咁先唔會兜圈。然後你會決定幾時件工作算做完，又或者幾時應該放棄。

呢啲步驟每一個都靠你嘅注意力去行。Agent 負責生成；而所有判斷都由你嚟提供。對於短期、探索性嘅工作嚟講，呢個方式好強勁，但佢有一個好硬嘅天花板：一旦你行開咗，進度就會即刻停低。你係喺最字面嘅意思上面，喺度湊住個 agent，每一個回合都要喺場，因為每一個回合都需要一個只有你先做得到嘅決定。

而家睇下一個 Loop 係點運作。Loop Engineering 就係將你喺腦入面做嘅嗰一串嘢，逐樣變成明確同埋寫低嘅形式，等一個系統可以代你去做。啲判斷唔會消失；佢哋只係由你嘅注意力裏面，搬咗去 Loop 嘅設計裏面。

分別唔係話一個 Loop 比你小心咁提示更加聰明。分別係喺於邊個去行呢個循環。一個 Loop 唔需要你都行得到，亦即係話佢可以喺你瞓覺嗰陣行，可以行成幾百次迭代，亦可以行嗰啲又長又磨人、又重複、悶到唔想用手去湊嘅工作。

我哋嚟建立呢種直覺。點解叫佢做 Loop？呢個名唔係一個比喻；佢係對個機制嘅描述。個系統會循環，佢會重複一個循環，而正正係呢種重複，先令到佢咁有價值。
