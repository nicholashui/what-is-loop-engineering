我哋就由最核心嘅諗法講起。Loop，即係循環，係由 loop engineer，即係循環工程師，所建立嘅一套統籌系統。佢會發掘或者觸發工作，喺隔離嘅環境入面開啟代理，驗證佢哋嘅輸出，喺多次運行之間保存狀態，跟住再決定係咪繼續、開啟子代理、定係停低。

而 Loop Engineering，即係循環工程，係一種實踐，講嘅係設計一套控制系統，去自動咁向 AI 編碼代理發出指令、觀察、驗證同迭代，而唔係靠人手一個回合一個回合咁去逐次提示佢。佢處於一條脈絡嘅最頂端，呢條脈絡由 Prompt Engineering 開始，經過 Context Engineering，再去到 Agent Harness Engineering。

呢條脈絡值得我哋逐個階段順住嚟講。Prompt Engineering，即係提示工程，係第一個階段。佢嘅實踐就係雕琢一條單一指令嘅措辭，包括用字、例子、輸出格式、角色設定，全部都係為咗引導模型，喺一個提示對一個回應之間，畀到一個好嘅答案。

Context Engineering，即係情境工程，係第二個階段。佢嘅實踐就係刻意咁去組織放入模型情境視窗嘅資訊，包括相關嘅檔案、檢索返嚟嘅文件、之前嘅對話、例子同指令，等模型可以根據啱嘅材料去推理。簡單講，佢係喺 Prompt Engineering 精挑細選嘅用字之上，再加入啱嘅資訊。

Agent Harness Engineering，即係代理框架工程，係第三個階段。佢嘅實踐就係為單一一次代理運行，建立一個可靠嘅執行環境，呢個環境就叫做 Harness。佢喺一個工作階段入面加入咗行動同回饋，意思即係工具、觀察到嘅結果，同埋多步驟嘅控制，全部都係建立喺精心整理過嘅情境之上。

咁就帶到我哋去講 Harness 本身。Harness，即係框架，係為單一一次代理運行而設嘅可靠執行環境。佢會供應工具畀代理，將工具嘅結果回饋返入代理嘅情境，管理一個多步驟工作階段嘅來來回回，並且令嗰次運行保持穩定。Loop 嘅統籌層，就係建立喺 Harness 之上。

跟住，講吓代理。AI Coding Agent，即係 AI 編碼代理，係一套以大型語言模型為基礎嘅系統，佢可以就一個軟件任務去推理，並且採取行動，包括執行命令、讀取同編輯檔案、檢查結果，而唔係淨係產生文字。喺一個 Loop 入面，AI 編碼代理通常會擔當 Generator Agent 嘅角色。

Generator Agent，即係生成代理，係 Loop 入面負責產生實際輸出嘅部件，譬如代碼、修改，或者其他成品。佢通常係一個 AI 編碼代理，例如 Claude Code 或者一個 Grok 代理。

Sub-agent，即係子代理，係 Loop 開啟出嚟嘅一個次級代理，用嚟處理工作入面一個聚焦嘅部分，譬如一個由主運行分支出嚟嘅專門任務。開啟子代理，係 Loop 喺決策點嗰三個結果之一，另外兩個就係繼續同停低。

接住，講吓一個 Loop 實際上係點樣運行。Iteration，即係迭代，係 Loop 嘅一次完整流程，包括發掘或者接收工作、運行一個代理、評估輸出、保存狀態，跟住再決定下一步做乜。一個 Loop 係透過好多次迭代去推進一個目標，而唔係靠單一一次運行。

Stopping Condition，即係停止條件，係一條會令 Loop 終止嘅規則。例子包括測試通過、Rubric 嘅分數達到某個門檻，同埋去到最大迭代次數，或者用盡咗預算。

Token Budget，即係代幣預算，係對 Loop 可以消耗嘅代幣數量，亦即係成本，所設嘅一個上限。緊嘅預算配合停止條件，可以幫手減低代幣成本不受控咁增長嘅風險。

而家我哋逐個睇吓組成一個 Loop 嘅部件。Automation，亦都叫做 Trigger，即係自動化或者觸發器，係決定工作幾時同點樣進入 Loop 嘅部件。例子包括一個 cron 排程、一個喺有新議題時觸發嘅 webhook，同埋一個按需要呼叫嘅斜線式命令。

Evaluator，即係評估器，有時又叫做 Critic 評論者或者 Verifier 驗證器，係喺代理外部獨立咁檢查 Generator Agent 輸出嘅部件。佢可能係一套測試、一個做代碼審查或者評論嘅代理，又或者係一個 Rubric 評分器。佢就係令一個 Loop 可以判斷工作究竟係咪真係好，而唔係單靠代理自己嘅評估。

Rubric，即係評分準則，係一套明確、有評分嘅評估標準，用嚟判斷 Generator Agent 嘅輸出。一個 Evaluator 可以用 rubric 去畀分，而 rubric 分數嘅門檻又可以用嚟做停止條件。

State，亦都叫做 Memory，即係狀態或者記憶，係喺多次迭代之間保存進度嘅部件，咁就算任何一次單一運行結束，工作都唔會流失。例子包括一個 TODO file、一個草稿區、一塊專案看板，同埋磁碟上面嘅檔案。

Worktrees，亦都叫做 Isolation，即係工作樹或者隔離，係令每次代理運行都保持隔離同可以還原嘅部件。例子包括 git worktrees 同埋分開嘅分支，咁就令代理嘅工作可以進行，而唔會騷擾到主線嘅開發。

Connectors，即係連接器，係將 Loop 同外部系統整合埋一齊嘅部件。例子包括源碼控制平台、議題追蹤系統，同埋聊天平台。

Skills，亦都叫做 Knowledge，即係技能或者知識，係向代理提供可重用指引同標準嘅部件。例子包括一個技能檔案、一個代理指引檔案，同埋編碼標準。

跟住講吓一啲基礎嘅模式同前身。ReAct，呢個字係 Reason 加 Act，即係推理加行動，係一種將推理步驟同行動交替進行嘅代理模式。代理會諗吓下一步做乜，透過一件工具採取一個具體行動，觀察結果，再次推理，咁樣一直去到任務完成為止。佢係 Loop 嘅概念祖先，因為 Loop 就係將呢個推理加行動嘅循環，由單一一個代理對話入面抽出嚟，放入一套外部、可控制嘅系統。

Ralph Loop，亦都叫做 Ralph Wiggum 技巧，係 Loop Engineering 由佢演化出嚟嘅前身技術。佢係一個簡單嘅 shell 循環，會不斷咁餵畀代理新鮮嘅情境，再加上前一次運行嘅輸出同錯誤，跟住重複運行，直到任務完成為止。佢展示咗喺一個循環入面運行代理、再將結果餵返入去呢個核心諗法，但係就無隔離、無一個獨立嘅評估器，亦都無保存狀態。

Manual，即係人手提示，或者話逐個回合提示，係同 AI 編碼代理合作嘅傳統方式，即係由一個人發出一條指令，讀吓結果，再發出下一條指令，逐步咁監督每一步。Loop Engineering 就係用一套會自己去提示同迭代代理嘅系統，去取代呢種親力親為嘅監督。
