喺呢一章，你會了解到呢本書想教你啲乜嘢、係寫畀邊啲人睇，仲有你睇完之後會達到嘅具體學習目標。你亦都會第一次用最淺白嘅方式，認識吓乜嘢係 Loop（迴圈），佢同你親手逐句去 prompt 一個 AI coding agent 有乜分別，以及後面幾章會逐一深入拆解嘅核心組件。

咁我哋就由「點解會有呢本書」講起。大部分工程師接觸 AI coding agent 嘅方式都係差唔多：一個對話框、一句 prompt、一個回應，跟住再嚟多一句 prompt。你一直坐喺司機位，一個回合一個回合咁去操控個 agent。咁樣係得嘅，但係佢會為你能夠完成嘅嘢設下一個好硬嘅上限，因為你嘅注意力會變成樽頸。每行前一步，都要你睇返個輸出、判斷有乜嘢唔妥，再打出下一句指示。

Loop Engineering（迴圈工程）就係一門專門去拆走呢個樽頸嘅學問。你唔再係一個回合一個回合咁手動去 prompt 個 agent，而係設計一個細細嘅控制系統，即係一個 Loop，由佢嚟幫你 prompt 個 agent。呢個 Loop 會自己發掘要做嘅工作、喺一個隔離嘅工作區度行個 agent、檢查結果、將問題反饋返入去、記住發生過嘅嘢，再自己決定係繼續做落去定係停。你嘅工作就由打 prompt，轉移去設計嗰個負責 prompt 嘅系統。

呢本書會教你點樣去思考、去推敲，同埋去建構呢個系統。佢假設你已經喺用家嘅層面用緊 AI coding agent，然後由呢一點出發，帶你去設計一啲你可以放心交託長時間、反覆工作嘅自主 Loop。

咁，你睇完之後究竟實際上識做啲乜嘢呢？每一個目標都由一章或者幾章具體去交付，所以你隨時都可以將一個目標追溯返去達成佢嘅內容。

第一，你會識得解釋乜嘢係 Loop，以及佢同手動 prompt 有乜分別。呢個由第三章交付，而本章稍後亦都會先預覽吓。

第二，你會識得將 Loop Engineering 放返入佢嘅脈絡入面，將佢同 ReAct 模式扣連起嚟，ReAct 即係 Reason 加 Act，亦即推理加行動。你會明白呢個範疇係點樣由 prompt engineering（提示工程），經過 context engineering（脈絡工程），再到 agent harness engineering（代理框架工程），最後發展到 Loop Engineering，以及一個 Loop 同 ReAct 模式嘅關係。呢部分喺第四章講。

第三，你會識得講返呢個名詞嘅起源，以及佢嘅前身，即係叫做 Ralph Loop 嘅技巧。你會知道呢個名詞由邊度嚟、係邊個塑造佢，以及嗰個簡單嘅 Ralph Loop 技巧，係點樣演變成可以用喺生產環境嘅 Loop Engineering。呢部分喺第五章。

第四，你會識得辨認同解釋每一個 Loop 組件嘅用途。你會識得描述全部八個基本構件，即係自動化同觸發器、隔離、generator agent（生成代理）、evaluator（評估器）、狀態同記憶、技能同知識、連接器，以及停止條件，而且每一個都舉到一個具體例子。呢部分由第六章交付。

第五，你會識得由頭到尾追蹤成個 Loop 嘅生命週期，以及佢嘅決策點，沿住個有序嘅流程，由觸發工作一路行到去決定係繼續、係衍生一個 sub-agent（子代理），定係停低。呢個就係第七章。

第六，你會識得建構一個最簡單嘅 Loop，再將佢演化成一個可以用喺生產環境嘅版本。你會由一個簡單嘅 bash 式迴圈開始，逐步將佢養大成一個有隔離、有獨立 evaluator、有持久化狀態，同埋有明確停止條件嘅系統。呢個就係第八章嘅實作工夫。

第七，你會識得判斷幾時應該用 Loop，並且權衡佢嘅好處。你會認得到啲真正值得用 Loop 嘅工作類別，例如重構、功能實作同除錯，而且你會講得出點解。呢個就係第九章。

第八，你會識得預先估到 Loop 嘅風險，並且採取具體嘅緩解措施。你會學識點樣防範失控嘅成本同低質素嘅輸出，亦都明白喺邊啲位，人嘅工程判斷力依然係不可或缺。呢個就係第十章，而結尾嘅綜合總結就喺第十一章。

如果你呢八樣嘢都做得到，你就已經由「操作一個 AI coding agent」，提升到「工程設計嗰個幫你操作佢嘅系統」。

喺我哋行得更深入之前，我哋先建立一個初步嘅圖像。究竟乜嘢先至係一個 Loop 呢？後面幾章會好仔細咁鋪陳每一個概念，但係由一開始就帶住一個可用嘅心智模型，會好有幫助。有三條問題貫穿成本書，而呢一節就會每一條都答到啱啱好夠，令你定到位。

第一，乜嘢係 Loop。一個 Loop 係一個編排系統，係一啲少量嘅軟件同設定，佢會驅動一個 AI coding agent，經歷一輪又一輪嘅工作週期，而唔需要人喺每一步去 prompt。佢唔係個 agent 本身。個 agent 只係 Loop 嘅其中一部分。Loop 就係周圍嗰套機制，負責決定個 agent 應該做乜、行佢、判斷結果，再不斷重複，直到工作達到一個既定目標，或者撞到一個既定上限為止。呢個名好直接：個系統就係喺度兜圈。佢產出輸出、評估嗰個輸出，再用嗰個評估去驅動下一次迭代，一次又一次，就好似你自己親自監督個 agent 一樣，分別只在於：今次係由 Loop 嚟做監督。

第二，佢同手動 prompt 有乜分別。喺手動、逐個回合咁 prompt 嘅情況下，個控制系統就係人。你睇每一個回應、將目標放喺腦入面、判斷個輸出夠唔夠好，再砌出下一句 prompt。你一離開，進度就即刻停低。而用一個 Loop，呢個控制系統只係設計一次，跟住就自己行。你以前喺腦入面做嘅嗰啲決定，例如「呢個夠唔夠好」、「跟住應該發生乜嘢」、「我哋做完未」，全部都變成明確、寫死咗嘅規則：一個檢查輸出嘅 evaluator、一個記住進度嘅狀態檔案，以及一個結束整個運行嘅停止條件。實踐者嘅角色就由操作員，轉移去做系統架構師。你設計個 Loop，個 Loop 負責 prompt。

第三，核心組件。每一個 Loop，由最簡單嗰個去到最精密嗰個，都係由同一小組基本構件砌成。你會喺第六章逐一深入研究，但係喺呢度先介紹吓班角色，等本書其餘部分讀起上嚟成為一個連貫嘅整體。有自動化，又叫觸發器，佢決定工作幾時、用乜方式進入個 Loop，例如一個排程、一個事件好似新開咗一張 issue，又或者係你親手行嘅一個指令。有隔離，通常係透過 worktree 嚟做，佢令到每一次 agent 運行都各自分開、可以還原，咁樣啲錯誤就唔會污染到你嘅主工作區。有 generator agent，即係嗰個真正產出輸出嘅 AI coding agent。有 evaluator，佢透過行測試、審查程式碼，或者根據一個評分準則去打分，嚟核實個輸出。有狀態，又叫記憶，佢將進度跨越多次迭代咁保存落嚟，等個 Loop 記得做咗啲乜。有技能同知識，佢提供啲可重用嘅指引同標準，畀個 agent 去跟。有連接器，佢將個 Loop 同外部系統整合，例如版本控制、issue 追蹤工具，以及聊天平台。最後就係停止條件，即係結束個 Loop 嘅規則，例如測試全部通過、達到某個質素門檻，又或者撞到一個預算上限。

呢個圖像，暫時鬆鬆哋揸住就得喇。重點好簡單：一個 Loop 係由多個部件砌成；嗰啲部件取代咗你本來要親手做嘅判斷；而將佢哋設計得好，就正正係呢本書要教你嘅手藝。

我哋亦都睇吓本書其餘部分係點樣編排。各章節跟住一條刻意安排嘅軌跡。頭幾章建立「係乜」同「點解」：第二章講本書係寫畀邊啲人睇、第三章畀一個精準嘅學科定義、第四章講佢嘅脈絡同思想根源，而第五章就講佢嘅起源。中間幾章拆解各個部件以及佢哋之間點樣互動：第六章講嗰八個組件，第七章講由頭到尾嘅生命週期。實作嗰章教你點樣建構一個出嚟，就係第八章。結尾幾章涵蓋幾時應該用 Loop、佢嘅好處同風險，以及一個最終嘅綜合總結，分佈喺第九章到第十一章。最後仲有詞彙表同參考資料一節，將本書補得完整。你可以一路順住讀落去當作一個完整課程，又或者如果你鍾意，亦都可以直接跳去某個特定主題。

咁我哋就總結返啲主要概念。Loop Engineering 係一種實踐，係去設計一個控制系統，即係一個 Loop，由佢自主咁去 prompt 同編排 AI coding agent，而唔係一個回合一個回合咁手動去 prompt 佢哋。一個 Loop 係嗰個編排系統，唔係個 agent；個 agent 只係佢入面其中一個組件。佢同手動 prompt 嘅關鍵分別，就在於係邊個控制成個週期：用一個 Loop，人本來要親手做嘅判斷就會變成明確、寫死咗嘅規則，咁就將你嘅角色由操作員轉移去做系統架構師。每一個 Loop 都係由同一組核心組件砌成：自動化同觸發器、隔離、一個 generator agent、一個 evaluator、狀態同記憶、技能同知識、連接器，以及一個停止條件。而呢本書有八個明確嘅學習目標，每一個都由特定章節去交付，帶你由理解一個 Loop，一路去到建構同判斷一個 Loop。
