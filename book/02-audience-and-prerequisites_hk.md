# 02. 讀者對象與先備知識

> **本章內容：** 你會了解本書是為邊個而寫、它假設你已經具備嘅背景知識，
> 以及它點樣處理任何超出呢啲背景嘅概念。睇完本章之後，你就可以喺投入時間之前，
> 確認呢啲內容係咪啱你嘅程度。

## 本書適合邊啲讀者

本書係寫畀**已經喺使用 AI 編程代理（AI coding agent），但仲未習慣自主地統籌（orchestrate）佢哋嘅軟件工程師同技術從業員。**

如果你曾經開過一個 AI 編程助手、描述你想做嘅改動、睇佢嘅回應，然後再打多個跟進提示（prompt）去引導佢，咁你正正就係本書設想嘅讀者。你習慣咗做迴圈（Loop）入面嗰個人——睇每一個結果、判斷邊度出錯、再親手提示下一步。你仲未做過嘅，就係將呢個監督角色交畀一個你親自設計嘅系統。

呢個正正就係本書想幫你跨越嘅一步。本書唔會教你 AI 編程代理係乜嘢、又或者點樣好好咁提示佢；它假設你已經識做呢啲嘢。它會帶你由**逐回合操作**一個代理，去到**設計嗰個替你操作佢嘅迴圈（Loop）**。重心係代理周邊嗰個統籌系統——發掘工作、喺隔離環境運行代理、驗證結果、記住進度，以及決定要繼續定停止——而唔係寫一個好提示嘅技巧。

以下幾類讀者喺呢度會有賓至如歸嘅感覺：

- **在職工程師**：日常依賴 AI 編程代理，想由逐回合睇住佢嘅瑣碎工作中,
  重新攞返自己嘅專注力。
- **技術主管同平台工程師**：正評估自主代理工作流程值唔值得喺自己團隊採用，
  希望對當中嘅好處**同埋**風險有一個老實嘅全貌。
- **愛鑽研嘅人同早期採用者**：曾經喺 shell 腳本入面跑過一個簡單嘅代理迴圈，
  想了解點樣將它演化成一啲穩陣可靠嘅嘢。

如果你從未用過任何 AI 編程代理，你一樣可以跟住睇，不過只要花少少時間親手試下其中一個，每一章都會喺你心入面落得更穩。

## 你應該已經懂得啲乜

為咗令本書聚焦喺 Loop Engineering（迴圈工程）本身，它假設你具備少量先備知識。當中冇任何高深嘅嘢；全部都係一個在職工程師通常已經有嘅背景。

### 版本控制（Git）

你應該熟悉版本控制嘅基本概念，尤其係 **Git**：commit、branch（分支），以及「工作可以喺一條 branch 上隔離、之後再 merge（合併）返」嘅諗法。迴圈好倚重隔離——將每一次代理嘗試喺一個獨立嘅地方運行，令一個壞結果永遠唔會污染你嘅主程式碼——而本書會用 branch 同 worktree 等 Git 概念去解釋呢種隔離係點樣達成。（*worktree* 係 Git 嘅一個功能，可以同一時間將多過一條 branch 簽出（checkout）到唔同嘅工作目錄；本書喺有需要嘅地方會提你一句，所以對 branch 有少少基本認識就足夠起步喇。）

### 命令列使用

你應該識喺終端機（terminal）入面讀同跑命令。書中嘅實作例子會由一個簡單嘅 shell 迴圈開始，再逐步建立上去，所以假設你睇得明一段短嘅 `bash` 腳本——變數、一個迴圈、調用一個命令。你唔使係 shell 腳本嘅專家；你只需要對住終端機唔會驚,而且認得出嗰幾行喺度做緊乜。

### 同 AI 編程代理互動

你應該至少親手用過一個 AI 編程代理——例如提示佢寫或者改程式碼，再讀返佢產出嘅嘢。呢個係最重要嘅先備條件，因為成本書都係喺重新詮釋呢個你熟悉嘅經驗：你今日親手做嘅嘢，就係將來一個迴圈會替你做嘅嘢。你需要對「代理擅長乜、喺邊度會出錯、以及點解而家仲要有個人留喺迴圈入面」有一種切身嘅感受——咁你先至會體會到，要移走嗰個人究竟要乜嘢條件。

你**唔需要**以下任何一樣嘢：建立代理框架嘅往績、識用某個特定統籌工具或廠商、機器學習或者模型訓練嘅背景，又或者除咗讀得明實作章節入面嗰啲短而有註解嘅例子之外嘅任何特定程式語言。

## 本書點樣處理新概念

Loop Engineering 建基喺幾個概念之上——有啲你會識，有啲你可能未識。本書遵守一條硬規矩：**任何超出以上先備知識嘅概念，都會喺被倚賴之前先定義清楚。**

實際上，呢個意思係：

- **術語喺使用之前先介紹。** 當一章需要用到一個概念，例如 *Generator Agent（生成代理）*、*Evaluator（評估器）*、*Harness（執行框架）* 或者 *Stopping Condition（停止條件）*，它會喺第一次用到嗰處用淺白嘅語言解釋,而唔係假設你已經記住個定義。
- **縮寫第一次出現時會寫全。** 例如 *ReAct*（Reason + Act，推理 + 行動）模式喺第 04 章首次介紹時會寫出全名,然後本書先至用簡寫。
- **有一份詞彙表撐起成本書。** 每個關鍵術語喺[詞彙表](glossary_hk.md)入面都有一個單一、一致嘅定義集中收錄,所以就算你跳入後面嘅章節,你永遠可以查返個術語,而唔使去搵佢最初喺邊度解釋過。

目標係令你永遠唔會撞上一幅冇解釋過嘅行話高牆。如果你符合全部三個先備條件——版本控制、命令列,以及親手用 AI 編程代理嘅經驗——本書需要嘅其他一切,它都會喺過程中教返你。

## 重點摘要

- 本書係寫畀**已經喺使用 AI 編程代理、但仲未建立過系統去自主統籌佢哋嘅軟件工程師同技術從業員。**
- 它假設三個先備條件：熟悉**版本控制（Git）**、習慣**命令列使用**,以及親手**同 AI 編程代理互動**嘅經驗。
- 它**唔**假設你有建立代理框架嘅往績、識用任何特定工具或廠商,又或者有機器學習背景。
- 任何超出嗰啲先備條件嘅概念都會**喺使用之前先定義**,縮寫喺首次使用時會寫全,而[詞彙表](glossary_hk.md)就係術語嘅單一真相來源。
