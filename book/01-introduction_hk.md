# 01. 引言與學習目標

> **本章內容：** 你將會了解本書打算教授甚麼、目標讀者是誰，以及讀完之後你會
> 達成的明確學習目標。你亦會初步用淺白語言認識甚麼是 *Loop*（循環）、它與
> 親手逐輪向 AI 編程代理（AI coding agent）下提示有何分別，以及之後各章會深入
> 拆解的核心組件。

## 本書為何而寫

大多數工程師認識 AI 編程代理的方式都一樣：一個聊天框、一個提示、一個回應，
然後再下一個提示。你一直坐在駕駛位，逐輪去操控個代理。咁樣係行得通，但會為你
能夠完成嘅事設下一個硬性上限——你嘅注意力變咗做樽頸。每行前一步，都需要你
去閱讀輸出、判斷邊度出錯，再輸入下一條指示。

**Loop Engineering（循環工程）** 就係消除呢個樽頸嘅學科。你唔再逐輪親手向代理
下提示，而係設計一個細小嘅控制系統——一個 *Loop*——由佢代你向代理下提示。
個 Loop 會發掘工作、喺一個隔離嘅工作區運行代理、檢查結果、將問題反饋返入去、
記住發生過嘅事，並且自行決定要繼續定停低。你嘅職責由輸入提示，轉變為設計
嗰個負責下提示嘅系統。

本書會教你點樣去思考、推敲並建構嗰個系統。它假設你已經喺使用者層面用緊
AI 編程代理，並由呢一點出發，帶你去到設計可以信任、能夠處理長時間反覆工作嘅
自主 Loop。

## 你將能夠做到甚麼

讀完本書之後，你將能夠做到以下嘅事。每一個目標都由一章或多章具體交付，所以你
隨時都可以將一個目標追溯返去達成佢嘅內容。

1. **解釋甚麼是 Loop，以及它與人手下提示有何分別。**
   _由第 03 章交付（下文亦有預覽）。_
2. **將 Loop Engineering 放置於它的脈絡之中，並與 ReAct（Reason + Act，
   推理＋行動）模式建立關聯。** 理解這個領域如何由 prompt engineering
   （提示工程）成長，經過 context engineering（情境工程），再到
   agent harness engineering（代理框架工程），最後到 Loop Engineering——
   以及一個 Loop 如何與 ReAct 模式相關。_由第 04 章交付。_
3. **講述這個術語的起源以及作為前身的 Ralph Loop。** 知道這個術語從何而來、
   由誰塑造，以及簡單的「Ralph Loop」技巧如何演化成生產級的
   Loop Engineering。_由第 05 章交付。_
4. **辨識並解釋每個 Loop 組件的用途。** 描述全部八個構建模塊——
   自動化／觸發器（automation/trigger）、隔離（isolation）、
   生成代理（Generator Agent）、評估器（Evaluator）、狀態／記憶
   （state/memory）、技能／知識（skills/knowledge）、連接器（connectors）
   以及停止條件（stopping condition）——並為每一個舉出具體例子。
   _由第 06 章交付。_
5. **追蹤端到端的 Loop 生命周期及其決策點。** 走一遍由觸發工作到決定應否
   繼續、衍生子代理（sub-agent）或停止的有序流程。_由第 07 章交付。_
6. **建構一個最小的 Loop，並將它演進成生產級的 Loop。** 由一個簡單的
   bash 風格循環開始，將它成長為一個具備隔離、獨立評估器、持久化狀態以及
   明確停止條件的 Loop。_由第 08 章交付。_
7. **判斷何時應該應用 Loop，並權衡它們的好處。** 認識 Loop 能夠帶來回報的
   工作類別——重構、功能實作、除錯——並講清楚原因。_由第 09 章交付。_
8. **預見 Loop 的風險並應用具體的緩解措施。** 防範失控的成本和低質素的輸出，
   並理解喺邊度人類嘅工程判斷仍然不可或缺。_由第 10 章交付，並在第 11 章
   作收尾的綜合論述。_

如果你八項全部都做得到，你就已經由 *操作* 一個 AI 編程代理，進化到
*為你操作它的系統做工程設計*。

## 初步圖像：甚麼是 Loop？

之後嘅章節會仔細鋪陳每一個概念，但由一開始就帶住一個可用嘅心智模型會有幫助。
有三條問題貫穿成本書，而呢一節會就每一條都解答到剛剛足夠去為你定位。

### Loop 是甚麼

一個 **Loop** 是一個編排系統（orchestration system）——少量的軟件和配置——
它驅動一個 AI 編程代理經過反覆的工作循環，而毋須人類為每一步下提示。它並不是
代理本身。代理只是 Loop 嘅其中一 *部分*。Loop 是周邊的機制，負責決定代理應該
做甚麼、運行它、判斷結果，並重複進行，直至工作達到一個既定目標或者撞到一個
既定上限為止。

個名係字面意思：個系統會 *循環*。它產生輸出、評估嗰個輸出，再用評估去驅動
下一次迭代——一次又一次——就好似你親自監督個代理時會做嘅一樣，只不過今次係
個 Loop 去做監督。

### 它與人手下提示有何分別

喺 **人手、逐輪下提示** 嘅情況下，人類就係控制系統。你閱讀每一個回應、喺腦海中
緊握住目標、判斷輸出夠唔夠好，再撰寫下一個提示。你一抽身離開，進度就會即刻停低。

有咗 **Loop**，嗰個控制系統只需設計一次，之後就會自行運作。你以往喺腦海中所做
嘅決定——*呢個夠唔夠好？下一步應該發生甚麼？我哋完成咗未？*——就會變成明確、
經過編碼嘅規則：一個檢查輸出的評估器、一個記住進度的狀態檔，以及一個結束運行的
停止條件。實踐者嘅角色由操作員轉變為系統架構師。你設計個循環；個循環負責下提示。

### 核心組件

每一個 Loop，由最簡單到最精密，都係由同一套細小嘅構建模塊砌出嚟。你會喺第 06 章
深入研究每一個，但以下先列出呢班「角色」，等本書其餘部分讀起上嚟係一個連貫嘅整體：

- **自動化／觸發器（Automation / Trigger）**——決定工作 *何時* 以及 *如何* 進入
  Loop（例如，一個排程、一個事件如有新 issue，又或者一條你運行嘅指令）。
- **工作樹／隔離（Worktrees / Isolation）**——令每一次代理運行保持分隔且可逆轉，
  使錯誤唔會污染你嘅主工作區。
- **生成代理（Generator Agent）**——真正產生輸出嘅 AI 編程代理。
- **評估器（Evaluator）**——透過運行測試、審查代碼或者對照評分準則去核實嗰個輸出。
- **狀態／記憶（State / Memory）**——跨迭代地持久化進度，令 Loop 記住已經做過甚麼。
- **技能／知識（Skills / Knowledge）**——提供代理應該遵循嘅可重用指引及標準。
- **連接器（Connectors）**——將 Loop 與外部系統整合，例如版本控制、issue 追蹤器
  以及聊天平台。
- **停止條件（Stopping Condition）**——結束 Loop 嘅規則，例如測試通過、一個質素
  門檻，又或者一個預算上限。

暫且鬆動咁握住呢個圖像。重點只係：一個 Loop 係由 *多個部分砌成*，嗰啲部分取代咗
你本來要親手做嘅判斷，而將佢哋設計得好，正正就係本書要教嘅手藝。

## 本書其餘部分如何編排

各章遵循一條刻意安排嘅弧線。前面幾章確立 *是甚麼* 同 *為甚麼*：本書嘅目標讀者
（第 02 章）、對呢門學科嘅精確定義（第 03 章）、它嘅脈絡與思想根源（第 04 章），
以及它嘅起源（第 05 章）。中間幾章剖析嗰啲 *部分* 以及佢哋如何互動：八個組件
（第 06 章）同端到端嘅生命周期（第 07 章）。實踐章節示範俾你睇點樣 *建構* 一個
（第 08 章）。收尾幾章涵蓋 *何時* 使用 Loop、佢哋嘅 *好處* 同 *風險*，以及最後嘅
綜合論述（第 09–11 章）。一份詞彙表同一個參考資料部分為本書作結。

你可以由頭讀到尾當成一個完整課程，又或者用目錄跳去某個特定主題。

## 重點摘要

- **Loop Engineering** 係設計一個控制系統——一個 *Loop*——去自主地向 AI 編程代理
  下提示及作編排嘅實踐，而唔係逐輪人手向佢哋下提示。
- 一個 **Loop** 係嗰個編排系統，而唔係代理；代理只係當中嘅其中一個組件。
- 與人手下提示嘅決定性分別在於 *邊個控制嗰個循環*：有咗 Loop，人類本來會親手做嘅
  判斷會變成明確、經過編碼嘅規則，將你嘅角色由操作員轉變為系統架構師。
- 每一個 Loop 都係由同一套核心組件砌成：自動化／觸發器、隔離、一個生成代理、
  一個評估器、狀態／記憶、技能／知識、連接器，以及一個停止條件。
- 本書有八個明確嘅學習目標，每一個都由特定章節交付，帶你由理解一個 Loop，去到
  建構並判斷一個 Loop。
