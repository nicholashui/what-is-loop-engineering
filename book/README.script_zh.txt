欢迎你。这本书讲的是 Loop Engineering，也就是循环工程。在接下来的几章里，我们会从最基础的地方开始，一起去探索它。

那么，究竟什么是 Loop Engineering 呢？说到核心，它就是一套做法，用来设计一些系统，让系统自己自动地去给出提示、去观察、去验证，再不断地让 AI 编程代理重复改进，而不需要你事必躬亲、一个回合一个回合地领着它走。与其坐在那里逐句逐句地敲指令，你不如搭建一个小小的系统，帮你自动推动这个代理去做事。这就是我们会不断回过头来讲的核心想法。

后面的章节，是特意按照一条学习路线来安排的。我们会从基础和概念讲起，这样等你读到需要动手实践、更偏向亲自上手的那些内容时，你已经明白什么是一个 Loop，也明白它为什么会这样运作。你可以从头听到尾，把它当成一门完整的课程；或者哪个题目对你来说最重要，你就直接跳到那里。

我们会从一段引言开始，讲清楚这本书教些什么，以及你一路学下去会达到哪些目标。接着我们会聊聊这本书真正是写给哪些人看的，以及它假设你已经具备了哪些背景知识。再往后，我们会为这门学问下定义，仔细看看 Loop Engineering 和大部分人习惯的那种手动、逐个回合给提示的做法，有什么区别。

讲完之后，我们会追溯这一切是从哪里来的。我们会跟着它从提示工程演变成循环工程的过程，也会拆解它和一个叫 ReAct 的模式之间的关系。我们还会讲讲这个名字的由来，以及启发了它的一个更早期的技巧，也就是 Ralph Loop。

打好了基础之后，我们就会讲解构和流程。我们会深入看看组成一个 Loop 的八个基本部件，逐个拆开来讲，接着会走一遍 Loop 的生命周期，也就是从头到尾的整个流程，还会讲到那个很关键的决策点，就是这个 Loop 在那一刻要选择继续走下去、还是衍生出新的 Loop、还是停下来。

接下来，就是这本书最实用的核心部分。我们会一起搭建一个 Loop，从一个最精简的例子，到一个达到生产级水准的例子，都会一步一步走一遍，也会探索怎样去设定清晰的目标，以及合理的护栏。

最后，我们会讲判断和总结。我们会聊聊 Loop Engineering 真正的好处，以及它真正适合用来做哪些工作。我们也会老老实实地讲它的风险和局限，包括成本、输出质量偏低的问题，以及这些 Loop 到现在都还需要我们付出的判断力。最后我们会把所有内容综合到一起，把 Loop Engineering 定位成它真正的本质：一个小小的、设计得很好的控制系统。

一路学下去，你还会找到一些辅助材料，包括一份词汇表，作为这本书术语的唯一准则；也有一批从原始材料整理出来的参考资料和论述，每一条都做了标记，方便你自己去查证。

那么，我们现在就开始吧。
