# 03. 何謂 Loop Engineering（循環工程）

> **本章內容：** 你會學到 *Loop Engineering*（循環工程）以及它所產出的 *Loop*（循環）
> 的精確定義，清楚睇到呢套做法同「逐回合手動 prompt（提示）一個 AI 編碼代理」
> 有咩分別，並且建立一個牢固嘅直覺，明白 Loop 究竟係乜、點解佢配得起呢個名。
> 本章亦正正交付全書第一個、亦係最基礎嘅學習目標：能夠講解 Loop 係乜，
> 以及佢同手動 prompt 有咩分別。

## 精確定義

**Loop Engineering**（循環工程）係一種實踐：設計一個控制系統，去自主咁
prompt（提示）、觀察、驗證同迭代 AI 編碼代理 —— 而唔係逐回合手動咁 prompt 佢哋。

呢一句承載咗好多重量，所以值得逐部分拆開嚟講：

- **一個控制系統。** 你所建構嘅嘢係一個系統，而唔係單一個精明嘅 prompt。佢
  係少量嘅軟件同配置，職責就係令一個代理持續高效咁朝住目標工作。
- **去 prompt、觀察、驗證同迭代。** 呢四個動詞構成咗個循環。系統向代理發出
  指令（*prompt*）、留意代理產出咗咩（*觀察*）、檢查嗰啲輸出係咪真係夠好
  （*驗證*），再用結果去推動下一次嘗試（*迭代*）。
- **AI 編碼代理。** 個系統會統籌一個或多個 AI 編碼代理 —— 即係你今日已經會
  親手 prompt 嗰種助手。
- **自主咁。** 一旦你設計好咗，個系統就會自己行呢個循環，唔再需要由人去
  逐句撰寫每一條指令。

呢套實踐所產出嘅產物有佢自己嘅名。一個 **Loop**（循環）就係由 loop engineer
（循環工程師）所建構嘅統籌系統：佢會發掘工作、生成代理、驗證代理嘅輸出、
持久化狀態，並且決定要繼續定停低。喺全書入面，*Loop Engineering* 指嘅係嗰套
**學科** —— 即係一種思考同設計嘅方式 —— 而 *Loop* 指嘅係**你所建構嘅嗰件嘢**。
分清呢兩者，本書其餘部分就會保持清晰：你做嘅係 Loop Engineering；你最終會得到
一個 Loop。

有一點澄清可以避免一個常見嘅混淆：Loop **並唔係**代理。AI 編碼代理 —— 本書
稱之為 **Generator Agent**（生成代理），即係真正產出代碼同編輯嘅嗰個組件 ——
只係 Loop 嘅*其中一部分*。Loop 係包圍住代理嘅整套機制：決定代理應該做咩、
運行佢、判斷結果，然後重複。當本書講「Loop 決定」或者「Loop 驗證」嗰陣，
指嘅就係呢套包圍住嘅機制，而唔係入面嗰個代理。

## 對比：手動 prompt vs. 一個 Loop

要最快理解 Loop Engineering，就係將佢直接同佢所取代嘅嘢擺埋一齊比較。

### 手動、逐回合 prompt 係點運作

當你親手 prompt 一個 AI 編碼代理嗰陣，**你自己就係嗰個控制系統。** 諗一諗
喺一節工作入面你實際上做緊啲咩：

1. 你將目標記喺腦入面 —— 即係「完成」係咩模樣。
2. 你打出一個 prompt，描述下一步。
3. 你讀代理嘅輸出，再判斷佢究竟好唔好。
4. 如果錯咗，你就撰寫一個更正，然後再 prompt 多次。
5. 你記住已經試過咩，咁先唔會兜圈。
6. 你決定幾時工作算完成 —— 或者幾時應該放棄。

以上每一步都係靠*你嘅*專注力去行。代理負責生成；所有判斷都係你嚟提供。
呢套做法喺短時間嘅探索性工作上好強勁，但有一個硬上限：**你一行開，進度就
即刻停。** 你係喺最字面嘅意義上喺度「湊住」個代理 —— 每一個回合你都要喺場，
因為每一個回合都需要一個只有你先做到嘅決定。

### 一個 Loop 係點運作

Loop Engineering 將你喺腦海入面做嘅嗰一連串嘢，逐樣變成*明確而且編碼咗*嘅
形式，咁一個系統就可以代你去做。啲判斷並冇消失；佢哋只係由你嘅專注力裏面
搬咗出嚟，搬入咗 Loop 嘅設計入面：

| 喺手動 prompt 入面，*你*…… | 喺一個 Loop 入面，就變成…… |
|------------------------------|---------------------------|
| 將目標記喺腦入面 | 一份寫低咗嘅目標同評估準則 |
| 打出每一個下一步嘅 prompt | 一個自動觸發同 prompt 嘅步驟 |
| 判斷輸出好唔好 | 一個 **Evaluator**（評估器）（測試、一個評審代理、一份評分準則） |
| 記住已經試過咩 | 持久化喺磁碟上嘅**狀態／記憶** |
| 決定幾時停 | 一個明確嘅**停止條件** |

呢度正正係問題嘅核心，亦值得用最直白嘅方式講出嚟：**你唔再做湊仔保母，
而係建構嗰個會去 prompt、觀察、驗證同迭代嘅 loop。** 你嘅角色由*操作員* ——
即係嗰個打下一條指令嘅人 —— 轉變成*系統架構師* —— 即係嗰個設計咗「個打指令
嘅系統」嘅人。你嘅判斷係一次過、預先喺設計入面花費，而唔係一個又一個回合
持續咁花。

呢個分別並唔係話 Loop 比你細心 prompt「更聰明」。個分別在於*邊個去行個循環*。
一個 Loop 唔使你都行得到，即係話佢可以喺你瞓覺嗰陣行、可以橫跨幾百次迭代去
行，亦可以喺嗰啲又長、又磨人、又重複到唔值得用手親自湊嘅迭代任務上去行。

## 建立直覺：點解叫「Loop」？

呢個名唔係一個比喻；佢係對個機制嘅描述。個系統會**循環**（loop）—— 佢重複
住一個循環 —— 而正正係呢種重複，先令佢有價值。

將個循環想像成一個一次又一次咁轉嘅圓圈：

```text
        ┌─────────────────────────────────────────┐
        │                                           │
        ▼                                           │
   ┌─────────┐   ┌─────────┐   ┌──────────┐         │
   │ prompt  │──▶│ 觀察    │──▶│  驗證    │─────────┘
   │ (代理   │   │ (讀取   │   │ (好唔好? │   迭代:
   │  行動)  │   │  輸出)  │   │      )   │   將結果
   └─────────┘   └─────────┘   └──────────┘   餵返入去
                                     │
                                     ▼
                              ┌────────────┐
                              │  完成?     │  ── 係 ──▶ 停止
                              └────────────┘
```

每一次兜過個圓圈，Loop 都會 prompt 代理、觀察返嚟嘅嘢、對住目標驗證，然後
做一個決定：如果工作仲未夠好，佢就將結果 —— 包括任何錯誤或者未通過嘅檢查
—— 餵返入去，再兜多一次；如果工作達到目標（或者撞到上限），佢就停。

呢個正正就係你喺手動工作節中喺腦海入面行嘅嗰個循環。你 prompt、你讀、你
判斷、你更正、你重複，直到你滿意為止。Loop Engineering 只不過係將呢個循環
由你嘅腦海裏面提取出嚟，變成一個可以自己行嘅系統。一旦你清楚睇到呢個循環，
成套學科就跟住而嚟：之後各章所講嘅一切，都係關於點樣將呢個圓圈嘅每一部分
設計好 —— 工作點樣進入個圈、代理點樣喺入面安全咁運行、驗證步驟點樣決定
「夠好」、進度點樣喺回合之間記住，以及 Loop 點樣知道幾時應該停。

緊握住呢幅圖像。一個 Loop 就係一個會自己轉嘅圓圈，而 Loop Engineering 就係
設計一個值得任由佢轉嘅圓圈嘅手藝。

## 重點摘要

- **Loop Engineering** 係一種實踐：設計一個控制系統，去*自主咁* prompt、觀察、
  驗證同迭代 AI 編碼代理，而唔係逐回合手動咁 prompt 佢哋。
- 一個 **Loop** 就係呢套實踐所產出嘅嘢：嗰個會發掘工作、生成代理、驗證輸出、
  持久化狀態，並決定要繼續定停低嘅統籌系統。Loop *並唔係*代理 ——
  **Generator Agent** 只係佢入面嘅其中一個組件。
- 同手動 prompt 嘅決定性對比在於*邊個去行個循環*。喺手動 prompt 入面**你**
  就係控制系統，逐回合提供所有判斷；喺一個 Loop 入面，嗰啲判斷變成咗明確、
  編碼咗嘅部分 —— 一個目標、一個 **Evaluator**、持久化嘅狀態、一個停止條件
  —— 令個系統唔使你都行到個循環。
- 直白咁講：**你唔再做湊仔保母，而係建構嗰個會去 prompt、觀察、驗證同迭代
  嘅 loop。** 你嘅角色由*操作員*轉變成*系統架構師*。
- 個名係字面意思：一個 Loop 會*循環*，重複住一個 prompt → 觀察 → 驗證 →
  迭代嘅循環，直到工作達到目標或者撞到上限為止。嗰個重複住嘅圓圈，就係
  你要帶住行落全書其餘部分嘅心智模型。
